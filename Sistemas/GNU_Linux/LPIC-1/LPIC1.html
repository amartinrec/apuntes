LPIC - Capitulo 1 - Herrramientas de la linea de comandos

Definición

  • Consolas(Shells)de Linux son los programas que reciben e interprentan los comandos de tipo texto.

Tipos de consolas predeterminadas

  • Consola interactiva predeterminada
    • Es un programa que utiliza el usuario para : Introducir comandos, Ejecutar programas, Ejecutar Scripts, ...
  • Consola predeterminada del sistema
    • Linux utiliza para ejecutrar scripts de consola del sistema generalmente durante el inicio.

Comandos internos y externos

  • La mayor parte de los comandos de linux son externos, esto es, son independientes de la consola, sin embargo, hay un conjunto de comandos internos de la consola.
  • Los comandos internos son los integrados en la consola, son los responsables de llevar a cabo algunas tareas comunes.
  • Para determinar si un comando es interno se emplea el comando type seguido del comando del cual queremos saber si es interno. Con la opcion -a muestra si es interno y externo.
  • Ejemplos
    • Comandos internos
      • Ejemplo : $type cd
      • cd is a shell builtin (cd es una orden interna del shell)
    • Comandos externos
      • Ejemplo : $type bash
      • bash is /bin/bash
    • Comandos externos que hace lo mismo que un comando interno
      • Ejemplo: $type -a pwd
      • pwd is a shell builtin (pwd es una orden interna del shell)
      • pwd is /bin/pwd

Historia de las Command Shells

  • sh
    • Was written by Steve Bourne at AT&T in 1977
    • Often known as the Bourne Shell.
    • All other shells are descended from it in some fashion and it is available on all systems that have a UNIX bloodline.
  • csh
    • was written by Bill Joy at UC Berkeley and released in 1978.
    • The internal syntax is quite different than sh and is designed to resemble the C programming language, and hence the name.
  • tcsh
    • was originally developed by Ken Greer at Carnegie Mellon University in the late 1970's;
    • the t in tcsh stands for TENEX, an operating system that was used on some DEC PDP-10's.
    • It has many additional features as compared with csh and on virtually all modern systems csh is just a link to tcsh.
  • ksh
    • was written by David Korn at AT&T and appeared in 1982
    • often known as the Korn shell.
    • It was designed to be a major upgrade to sh and is backward compatible with it, and brings in some of the features of tcsh, such as command line history recall. This shell has long been a favorite of many system administrators.
  • bash
    • is a product of the GNU project and was created in 1987.
    • It was designed as a major upgrade of sh; the name stands for Bourne Again Shell.
    • It has full backward compatibility with sh and partial compatibility with ksh.
    • Consola por defecto mas común para cuentas de usuario.
  • zsh
    • Evolucion de la consola Korn

Observaciones

  • sh es un link a bash, csh es un link a tcsh
  • El archivo /bin/sh es un enlace simbolico a la consola por defecto del sistema, en Linux suele ser /bin/bash, en Ubuntu apunta a /bin/dash
  • On all Linux systems sh is just a link to bash, but scripts which are invoked as sh will only work without the bash extensions. A similar relationship exists between csh and tcsh.
