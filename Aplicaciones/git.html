Clase práctica de Git y Github

Modalidad 1: Empiezo a trabajar en local (en mi maquina)

  • Configuración de nuestro git

    1. $git config --global user.name <nuestro nombre> para establecer el nombre por defecto que se usa git cuando hacermos un commit
    2. $git config --global user.email <nuestro email> para establecer el email por defecto que se usa git cuando hacermos un commit

    Sirve para que los commit aparezcan con el usuario y email definido. Cuando haces un $git log se puede apreciar que ha puesto "Author: <nuestro nombre> <nuestro email> " en :
    commit a63a5e0c5c2233ca59eb81c38083b854851c191b
    Author: Ángel Martín <amartin@gmail.com>
    Date: Mon May 30 05:19:50 2016 +0200

  • Creación de la carpeta del proyecto y del repositorio local

    1. Creamos una carpeta local donde vamos a ir almacenando los ficheros de nuestro proyecto <carpeta del proyecto> y creamos un <fichero> en la <carpeta del proyecto>.
    2. Creacion del repositorio local usando el comando $git init
      Sabemos que se ha creado un repositorio porque en la <carpeta del proyecto> aparecerá un directorio ".git" donde se almacena la información del repositorio.
    3. Añadir los ficheros al repositorio

  • Proceso

    1. Añadir el fichero a la zona de index (staging area) usando el comando $git add <fichero>
    2. Para saber el estado de mis ficheros empleamos el comando $git status
    3. Hacer commit empleando el comando $git commit -m "mensaje" donde "mensaje" es que se asocia al commit.
    4. Para saber el estado de mis ficheros empleamos el comando $git status

  • Subir el proyecto a un repositorio a GitHub

    1. Crear un usuario en GitHub
    2. Vincular el respositorio remoto en el repositorio local con el comando $git remote add remoto -url del repositorio remoto -
    3. Listamos todos los remotos con el comando git remote -v
    4. Empujamos los cambios a GitHub con el comando push desde el repositorio local al repositorio remoto.$git push remoto master

Los tres estados de los archivos

Git tiene tres estados principales en los que se pueden encontrar tus archivos:

  • Confirmado (committed): Confirmado significa que los datos están almacenados de manera segura en tu base de datos local.
  • Modificado (modified): Modificado significa que has modificado el archivo pero todavía no lo has confirmado a tu base de datos.
  • Preparado (staged): Preparado significa que has marcado un archivo modificado en su versión actual para que vaya en tu próxima confirmación.

Esto nos lleva a las tres secciones principales de un proyecto de Git:

  • El directorio de Git (Git directory): El directorio de Git es donde Git almacena los metadatos y la base de datos de objetos para tu proyecto.Es la parte más importante de Git, y es lo que se copia cuando clonas un repositorio desde otro ordenador.
  • El directorio de trabajo (working directory): El directorio de trabajo es una copia de una versión del proyecto. Estos archivos se sacan de la base de datos comprimida en el directorio de Git, y se colocan en disco para que los puedas usar o modificar.
  • El área de preparación (staging area): El área de preparación es un sencillo archivo, generalmente contenido en tu directorio de Git, que almacena información acerca de lo que va a ir en tu próxima confirmación. A veces se le denomina índice, pero se está convirtiendo en estándar el referirse a ella como el área de preparación.

El flujo de trabajo básico en Git es algo así:

  1. Modificas una serie de archivos en tu directorio de trabajo.
  2. Preparas los archivos, añadiendolos a tu área de preparación.
  3. Confirmas los cambios, lo que toma los archivos tal y como están en el área de preparación, y almacena esas instantáneas de manera permanente en tu directorio de Git.

Si una versión concreta de un archivo está en el directorio de Git, se considera confirmada (committed).Si ha sufrido cambios desde que se obtuvo del repositorio, pero ha sido añadida al área de preparación, está preparada (staged). Y si ha sufrido cambios desde que se obtuvo del repositorio, pero no se ha preparado, está modificada (modified).

Comandos

  • Historico de las confirmaciones (commit) realizadas se puede ver con $git log
  • Mostrar los repositorios remotos:
    $git remote
  • Mostrar los repositorios remotos con su URL asociada:
    $git remote -v
  • Añadir repositorios remotos:
    $git remote add [nombre]
    Ejemplo:
    $ git remote add apuntes git://github.com/amartinrec/apuntes.git
  • Renombrar repositorios remotos:
    $ git remote rename <nombre_actual> <nuevo_nombre>
  • Eliminar repositorios remotos:
    $ git remote rm *nombre_actual*
  • Recibiendo de tus repositorios remotos:
    $ git fetch [remote-name]
  • Listar las propiedades de Git:
    $git config --list
    En Ubuntu crea un fichero /home/<usuario>/.gitconfig donde almancena los valores de las propiedades
  • Establecer nombre de usuario:
    $git config --global user.name "Nombre"
  • Establecer direccion de correo electronico:
    $git config --global user.email "direccion@correo"
  • Ayuda:
    $git help <comando>
  • Ayuda:
    $git <comando> --help
  • Inicializando un repositorio en un directorio existente:
    $git init
