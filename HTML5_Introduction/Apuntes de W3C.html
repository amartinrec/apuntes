Elementos semánticos

Separando contenido y estilo

Cuando escribimos en lenguaje HTML, es importante separar contenido y estilo. El estido deberia estar bien guardado en hojas de estilo en cascada (CSS)

Comparativa entre <b> y <strong>

Bold es un estilo que hace las letras más tupidas por lo que destaca entre otro texto pero no tiene un significado semántico

Strong es una indicación de como algo debería ser. Representa importancia, seriedad o urgencia del contenido.

Compartiva entre <i> y <em>

Italics es un estilo que inclina el texto.

Emphasis es una indicación para emfatizar el contenido.

Observación: Se le ha querido dar un nuevo uso a <i> en HTML5 que consiste en usarlo para expresar una voz o modo diferente, por ejemplo insertar frases en otros idiomas, para ello empleamos (<i lang="fr">À la carte<i>)

Estructura de página web

  • header: El header puede ser el punto de partida para todo el documento o secciones individuales. Contiene normalmente la introducción
  • nav: Se refiere a la navegación. Puede contener un conjunto de links de navegación, como una tabla de contenidos de un libro.
  • section: Secciones del documento. Por ejemplo, un documento acerca de plantas podría tener varias secciones bajo encabezamientos como "perennes", "caducas", etc.
  • article: Se refiere a contenido independiente. Si un articulo se extrae del documento, este no pierde el sentido.
  • aside: Se refiere a contenido al margen del contenido principal. Por ejemplo, en un articulo acerca de arquitectura de la region de Barcelona, se podria crear un aside informando sobre la región de Barcelona.
  • footer: autor, copyrights, información de contacto

Otros elementos semánticos

  • details: Una forma de proporcionar información adicional que el usuario puede mostrar o ocultar.
  • figcaption: Usado para proporcionar una explicación de una imagen. Se emplea junto con la etiqueta <figure>
